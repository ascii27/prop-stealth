# PropStealth UI Design Spec

**Date:** 2026-04-19
**Status:** Approved

---

## Visual Direction

**Style:** Clean & Professional — white backgrounds, green accents (#059669), inspired by Stripe/Notion.
**Typography:** System font stack (system-ui, -apple-system, sans-serif).
**Color palette:**
- Primary: #059669 (green — trust, money, growth)
- Background: #ffffff, #f9fafb, #f8faf8 (sidebar)
- Text: #111827 (headings), #374151 (body), #6b7280 (secondary), #9ca3af (muted)
- Attention: #fffbeb / #fde68a (amber — needs attention)
- Danger: #fef2f2 / #fecaca (red — vacancies, violations)
- Info: #eff6ff / #bfdbfe (blue — agent proposals, tenant tags)
- Purple: #7c3aed (HOA tag)

## Layout

**Navigation:** Left sidebar, always visible. Logo at top, nav items below, settings/actions at bottom separated by a divider.

**Two distinct app experiences** based on role selected at signup:
1. **Owner view** — task-centric, focused on what's happening with their properties
2. **Agent view** — client portfolio overview, tenant pipeline, help requests

### Owner (Dana) Screens

#### Activity Feed (Home)
- Greeting: "Good morning, Dana" with count of attention items
- "+ Evaluate Tenant" quick action button
- **Needs Your Attention** section at top:
  - Tenant proposals from agent (blue background) — links to Tenant Eval results
  - HOA violations with deadlines (amber background) — links to Inbox Agent
  - Tenant requests (amber background) — links to Inbox Agent
- **Timeline** below, grouped by day (Today, Yesterday, etc.):
  - Vertical timeline with green dots (today) and gray dots (older)
  - Each entry: title, property/context, timestamp
  - Covers inbox processing, tenant evaluations, classified emails

#### Inbox Agent
- Header: "Inbox Agent" with "Last scanned: X min ago" badge
- **Property filter tabs** across top: "All Properties" | specific property addresses
- Content organized by property, then by theme within each property
- **Theme groups** with color-coded dot + uppercase label + email count:
  - Tenant (blue #2563eb)
  - HOA (purple #7c3aed)
  - Bill (green #059669)
  - Other (gray #6b7280)
- Each email card shows:
  - Sender name, unit/property, timestamp
  - **"Key points:"** AI-generated summary (2-3 sentences)
  - **"Auto-respond"** button (green, opt-in per email) — only shown where a reply makes sense
  - **"Expand"** button — reveals full email content
- No auto-generated replies by default. User must click "Auto-respond" to trigger AI draft.
- Informational emails (bills, tax notices) do not show Auto-respond button.

#### Tenant Evaluation
Two states:

**Upload flow (New Evaluation):**
- Form: Applicant Name, Property (dropdown), Document upload (drag & drop)
- Uploaded files listed with name and size
- "Run Evaluation" button + "Cancel"
- FCRA disclosure note at bottom

**Results view:**
- Header: applicant name, property, evaluation date
- Large score badge (e.g., 82/100) with colored border (green/yellow/red)
- Recommendation badge: "Recommended — Low Risk" (green), "Review Carefully" (yellow), or "High Risk" (red)
- "AI-generated and advisory" disclaimer
- **Score Breakdown** — 4 cards in a grid:
  - Income (with rent ratio)
  - Employment (with tenure)
  - Rental History (with notes)
  - Credit Check (via TransUnion)
- **AI Summary** — narrative paragraph explaining reasoning with specific evidence
- **Evidence** — list of source documents and external reports, each clickable
- **Action buttons:** Approve Tenant (green), Decline (red), Ask Agent for Help (gray)
  - "Ask Agent for Help" bridges to the agent (Priya) — this is how Dana escalates

#### Properties (standard CRUD)
- List of properties with address, type, unit count, occupancy status
- Add/edit property form
- Per-property detail view with units, tenants, documents, key dates

#### Documents (standard CRUD)
- Per-property document vault
- Upload, auto-classify, search
- File list with type tags, upload date

#### Settings
- Gmail connection (OAuth)
- Gmail label configuration (which label the Inbox Agent monitors)
- Notification preferences
- Account settings

## Interaction Model

### Owner ↔ AI Agents
- AI agents (Inbox + Tenant Eval) work on behalf of the owner
- Owner reviews and approves agent outputs
- No autonomous email sending — always draft-for-approval via "Auto-respond" button
- Owner sees full activity log of agent actions

### Owner ↔ Real Estate Agent
- Owner can click "Ask Agent for Help" on tenant evaluations or from any screen to escalate to their agent
- Agent sees help requests in their dashboard
- Agent can run tenant evaluations and propose candidates to the owner
- Owner receives proposals in their Activity Feed and can review/approve
- Agent does NOT have visibility into the owner's inbox activity or agent communications

### Real Estate Agent ↔ AI Agents
- In MVP, AI agents only operate for clients (owners), not for the agent directly
- Agent uses the Tenant Evaluation agent to screen candidates for clients
- Future: agents could have their own AI agents for CRM, lead management, etc.

## Key Design Decisions

1. **Role toggle at signup, not separate URLs** — same page, different messaging and post-login experience
2. **Sidebar navigation** — scales for Phase 2 agents without redesigning nav
3. **Task-centric owner home** — Activity Feed, not property grid, because with only 2 agents a property-heavy UI feels empty
4. **Opt-in auto-respond** — Inbox Agent classifies and summarizes but does NOT draft replies unless user clicks "Auto-respond"
5. **Emails grouped by theme then property** — Tenant/HOA/Bill/Other with color-coded dots, organized under property headings
6. **"Ask Agent for Help"** as the bridge — owner escalates to their real estate agent from the tenant evaluation results view
7. **Agent dashboard is portfolio health, not operations** — vacancy rates, client status, tenant pipeline. Agent is not monitoring client inbox activity.
8. **Gmail label-based scoping** — user configures Gmail filters/labels themselves; our agent watches the specified label. No managed inbox in MVP.
